# {{PROJECT_NAME}} — Architectural Decisions

A running log of structural decisions, with the reasoning. Read this before making architectural changes — most "improvements" turn out to be re-opening a decision that was closed for a reason.

Each entry has three parts: **Decision** (what), **Reasoning** (why), and **Rejected alternatives** (what we considered and didn't pick). Entries are append-only, numbered sequentially (`D-001`, `D-002`, …).

Most ADRs are added by `/yuval-core:promote-to-code` when a brief is promoted. Manual ADRs (refactor decisions discovered mid-implementation) are also valid — log them with operation `adr` in `/code/wiki/log.md`.
