# Ideation — {{PROJECT_NAME}}

_Half-formed ideas, spitballs, "what if we" notes. One idea per file. Free-form bodies — no schema, no required structure. The point is to capture the shape of an idea early, before it's solid enough to become a backlog item or a brief._

**Filename convention:** `idea-<slug>.md` (kebab-case slug, unique within the folder).

**Lifecycle:**

- A file lives here as long as the idea is unresolved.
- When the idea matures into a backlog item, add a backlog entry referencing the file (or summarize and delete — author's choice).
- When the idea matures into a brief, link the brief from the file's footer (or delete the file once the brief subsumes it).
- When the idea is rejected, mark the file with `_Status: dropped — <reason>._` at the bottom and keep it as a record.

This folder is a **scratchpad with memory** — not a staging area for briefs. Don't optimize the file format; optimize for capture speed.
